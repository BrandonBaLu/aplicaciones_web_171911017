FORMULARIOS

INPUT
Etiqueta Descripcion Ejemplo de uso
<input> type=hidden representa cualquier cadena de texto arbitraria que no está pensada para ser vista o editada por el usuario. Los controles ocultos son especialmente útiles para enviar datos al servidor definidos por el autor, basados o no en la interacción con el usuario.

<input type="text" name="algodetexto">
<input> type=text Su valor predeterminado es text, si no se especifica este atributo

< input type="text" name="nombre" value="John" >
<input> accept Si el valor del atributo type es file, este atributo indica los tipos de archivo que el servidor acepta, de otra manera será ignorado. El valor debe ser una lista de tipos de contenido únicos, separados por coma < input type="file" name="archivosubido" >
<input>autocomplete Este atributo indica si el valor del control puede ser completado automáticamente por el navegador < input type="text" id="cc" name="cc" autocomplete="off" >
<input> autocorrect atributo no estándar que es usado para controlar si la autocorrección debe estar habilitada cuando el usuario está introduciendo/editando el texto de un input < input autocomplete="on" autocorrect="on" autocapitalize="on" type="text" name="textinput" placeholder="Enter a test" spellcheck="on" >
<input> autofocus atributo Booleano permite especificar que un control de formulario tenga el cursor cuando la página se carga, a menos que el usuario lo reemplace < input type="number" name="nombre" autofocus="autofocus"/ >
<input> formmethod Si el elemento input es un botón de tipo submit o image, este atributo especifica el método HTTP que el navegador usará para enviar el formulario < input type="submit" formmethod="get" value="enviar con GET" >
<input> formnovalidate Si el elemento input es de tipo submit o image, este atributo Booleano especifica que el formulario no será validado cuando se envíe. Si este atributo está especificado, reemplaza al atributo novalidate del formulario al que pertenece el elemento < input type="submit" formnovalidate="formnovalidate" value="Enviar sin validación" >
<input> formtarget Si el elemento input es de tipo submit o image, este atributo es el nombre o palabra clave que indica dónde mostrar la respuesta que se recibe después de enviar el formulario < input type="submit" formtarget="_blank" value="Enviar" >
<input> readonly atributo indica que el usuario no puede modificar el valor del control. El valor del atributo es irrelevante. De ser necesario el acceso lectura-escritura al valor, no se debe agregar el atributo "readonly < input readonly >
<input>name El nombre del control, el cual es enviado con los datos del formulario < input type="tipo" id="identificador" size="tamaño" name="nombre" value="texto por defecto"/ >

TEXTAREA
Etiqueta Descripcion Ejemplo de uso
placeholder pista acerca del contenido que se espera sea ingresado en el control, el cual será mostrado dentro de la caja de texto cuando ésta esté vacía < p > < textarea name="mensaje" placeholder="Comparte tu opinión con el autor!" >< /textarea >
name valor inicial como contenido del elemento este valor será usado para rellenar el control cuando el documento sea cargado y cuando el formulario sea restaurado. < p >< textarea name="comment" > Hola! Creo que estás completamente equivocado!< /textarea >< /p >
cols y rows establecer el número de caracteres que el control podrá mostrar al mismo tiempo, horizontal y verticalmente < p >< textarea name="descripcion" cols="40" rows="5" >< /textarea >< /p >
minlength y maxlength establecen la cantidades mínima y máxima de caracteres que el control puede recibir < p >< textarea name="biografia" minlength="20" maxlength="50" >< /textarea >< /p >
required atributo está presente, los navegadores que lo soportan impedirán el envío del formulario hasta que el control haya sido rellenado < p >< textarea name="comentarios" required >< /textarea >< /p>< /td>
